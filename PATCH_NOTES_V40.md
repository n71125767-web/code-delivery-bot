# V40 — визуал, корзина, прокси и рассылка

## Исправления
- Исправлена отмена мастера создания товара: `отмена`, `cancel`, `/cancel` теперь очищают состояние `CATALOG_V25_STATE` и не запускают создание по кругу.
- Фото категории больше не подставляется в карточки всех товаров. У товара показывается только его собственное фото/видео.
- Исправлен выбор прокси: бот больше не требует строго отдельный `fulfillment_type=proxyline` на этапе выбора страны, а использует активный товар из прокси-витрины.
- `/proxy_autofix` теперь создаёт привязки провайдеров так:
  - MTProxy и Premium → `proxyline`;
  - Standard и Residential → `proxys`.
- Добавлена поддержка provider type `proxys` в `ProductProvider`.
- Превью рассылки больше не добавляет лишнюю строку `📢 Предварительный просмотр рассылки`. Показывается ровно текст/фото/видео/документ, который получат пользователи.
- Рассылка теперь умеет отправлять фото, видео и документы с подписью.
- Если задача рассылки прерывается SIGTERM, статус помечается как `interrupted`, а не теряется молча.

## Новые функции
- Добавлена корзина:
  - кнопка `🛒 В корзину` в карточке товара;
  - раздел `🛒 Корзина`;
  - кнопки `+` и `−`;
  - ввод своего количества;
  - очистка корзины;
  - оформление до 5 позиций за раз.
- Добавлена кнопка `🤝 Стать партнёром`.
- Пользователь может отправить партнёрскую заявку обычным сообщением; заявка уходит администраторам с кнопками `принять / отклонить`.
- Переработан раздел `Мои заказы`: статусы стали читаемыми, добавлены эмодзи, количество, сумма и дата.
- Обновлены покупательские кнопки: `🛍 Каталог`, `🛒 Корзина`, `🤝 Стать партнёром`.

## Proxys API
Добавлены переменные окружения:

```env
PROXYS_ENABLED=0
PROXYS_API_KEY=
PROXYS_API_BASE_URL=
```

Важно: для `proxys` нужен совместимый API-адаптер/base URL с endpoint-ами `/ips-count/` и `/new-order/`. Если у вашего провайдера другой формат API, нужно подставить adapter URL или отдельно дописать адаптер под его документацию.

## Команды после деплоя

```text
/proxy_autofix 100 RUB
/proxy_markup 1.77
```

Если используете Standard/Residential через Proxys, добавьте в Render:

```text
PROXYS_ENABLED=1
PROXYS_API_KEY=ваш_ключ
PROXYS_API_BASE_URL=https://ваш_adapter_или_api
```
