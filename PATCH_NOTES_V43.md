# V43 — Proxyline balance handling fix

Исправлено поведение при ошибке поставщика Proxyline `Not enough money on balance`.

## Что изменено

- Покупка больше не уходит в бесконечные повторы выдачи при нехватке баланса поставщика.
- Ошибки поставщика, требующие действия админа, переводятся в статус `fulfillment_problem`.
- Покупатель получает нормальное сообщение: оплата принята, выдача временно задержана.
- Администратор получает уведомление с ID покупки и командой для повторной выдачи.
- Добавлена команда `/retry_purchase ID` для ручного повторения автовыдачи после пополнения баланса Proxyline/исправления поставщика.
- Recovery-loop больше не ретраит `delivery_failed`, если ошибка содержит `Not enough money`, `balance`, `not configured`, `not enough proxies` и похожие ручные проблемы.

## Как пользоваться

1. Пополните баланс Proxyline.
2. В Telegram от админа выполните:

```
/retry_purchase 3
```

Где `3` — ID покупки из уведомления/лога.
