# V73 Supplier / Number / Withdrawal Fix

Исправлено:

- Исправлена ошибка поставщика `value out of int32 range`: Telegram ID больше не используется как `suppliers.id`.
- Кнопка «📱 Номер» теперь настраивает товар как выдачу через поставщика: покупатель после оплаты выбирает сервис, затем поставщик получает запрос на номер, потом запрос на код.
- Добавлены типы выдачи через поставщика: «📱 Номер», «👤 Аккаунт», «🧩 Другое».
- Для готовых аккаунтов/ручной выдачи поставщик получает заявку сразу после оплаты и отправляет данные товара покупателю.
- Запрос кода поставщику после отправки номера сохранён.
- Вывод поставщика через кнопку теперь работает без `/withdraw`: после кнопки «Вывод» поставщик отправляет `сумма адрес` одним сообщением.
- Если авто-чек CryptoBot не создался, заявка остаётся на ручной модерации и админам уходит уведомление.
- Кнопки поставщика не перехватываются как номер/код/рассылка.
