# V50 — Admin/Supplier UI, Wallet Top-up, FAQ/Main Settings

## Исправлено

- Кнопка «Пополнить» в кошельке больше не ведёт в неизвестную команду.
- При входе в админку нижняя клавиатура теперь меняется на админскую, а не исчезает.
- Админ может вернуться в режим покупателя через «🏠 Главное меню».
- Отмена создания категории больше не запускает циклы мастеров.
- Автоматические эмодзи для новых категорий и системных прокси-товаров убраны.
- В категории админ видит список товаров с ID, названием и ценой, а не только счётчик.
- Панель поставщика упорядочена: заказы, товары, баланс, вывод, помощь, режим покупателя.
- Стандартные и резидентские прокси остаются привязанными к provider `proxys`.
- Названия провайдеров убраны из пользовательских описаний прокси.

## Добавлено

- Настройка главной страницы:
  - `/main_settings`
  - `/main_set ТЕКСТ`
- Настройка FAQ:
  - `/faq_set ТЕКСТ`
- Настройка сервисов для номеров:
  - `/number_services`
  - `/number_service_add Название`
  - `/number_service_remove Название`
- Пополнение внутреннего баланса через CryptoBot invoice:
  - кнопка «Пополнить» в кошельке;
  - `@send 10` или `/topup 10 USDT` в чате.
- Уведомление админам при зачислении пополнения.
- Поставщик может менять цену своих товаров:
  - `/supplier_price ID ЦЕНА [ВАЛЮТА]`
- Поставщики могут запрашивать вывод; комиссия вывода по умолчанию 2.5 USDT.
- Автовыплата через CryptoBot check, если метод доступен токену:
  - `WITHDRAW_AUTO_CRYPTOBOT=1`
  - `WITHDRAWAL_FEE_USD=2.5`
  - `WITHDRAW_PAYOUT_ASSET=USDT`

## Важно

Telegram Bot API не позволяет менять цвета кнопок напрямую. Визуал улучшен через структуру, подписи и эмодзи.

Внешние чеки CryptoBot, созданные не ботом, бот не может надёжно «поймать» сам. Поэтому для сценария `@send сумма` создаётся счёт CryptoBot, после оплаты деньги зачисляются на внутренний баланс магазина.
